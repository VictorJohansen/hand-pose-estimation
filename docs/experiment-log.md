# Experiment log

Tracks the serious training runs for the hand-pose-estimation project.
Each run is reproducible from the canonical FreiHAND split
(`SPLIT_SEED=42`, `validation_fraction=0.1`) and stores artifacts under
parallel `models/<run>/`, `logs/<run>/`, and `artifacts/<run>/`
directories.

## How to add a new run

1. Train, e.g. `python -m src.training.train_baseline --model-id baseline-model-1`
   or `python -m src.training.train_improved`. Outputs
   land in `models/<name>/best.keras`, `logs/<name>/config.json`, and
   `logs/<name>/history.json`.
2. Compute MPKE on the canonical validation split:
   `python -m src.evaluation.evaluate_run <name>`. Writes
   `artifacts/<name>/evaluation.json`.
3. Refresh the generated result summary:
   `python -m src.evaluation.report_summary`.
4. Regenerate numbered report figures:
   `python -m src.evaluation.report_figures`.

## Comparison

The current generated comparison is stored in `reports/result-summary.md`.
It is generated from the evaluation artifacts available under
`artifacts/<run>/evaluation.json`, so adding or removing evaluated runs does
not require maintaining a second hand-written result table.

## Report figures

Generated by `python -m src.evaluation.report_figures`:

- `report/figures/figure1.png`, `figure2.png`, etc.: numbered report figures.
- `docs/report-figure-captions.md`: descriptions associated with each
  numbered figure.

## Result summary

Generated by `python -m src.evaluation.report_summary`:

- `reports/result-summary.md`: generated comparison, ranking, relative
  delta, and run metadata tables.
