# Experiment log

Tracks the serious training runs for the hand-pose-estimation project.
Each run is reproducible from the canonical FreiHAND split
(`SPLIT_SEED=42`, `validation_fraction=0.1`) and stores artifacts under
parallel `models/<run>/`, `logs/<run>/`, and `artifacts/<run>/`
directories.

## How to add a new run

1. Train, e.g. `python -m src.training.train_baseline --model-id baseline-model-1`
   or `python -m src.training.train_improved`. Outputs
   land in `models/<name>/best.keras`, `logs/<name>/config.json`, and
   `logs/<name>/history.json`.
2. Compute MPKE on the canonical validation split:
   `python -m src.evaluation.evaluate_run <name>`. Writes
   `artifacts/<name>/evaluation.json`.
3. Append a `### <name>` section to this file with model, settings, and
   notable observations.
4. Refresh the comparison table:
   `python -m src.evaluation.comparison <run-a> <run-b> ...`.

## Runs

### `baseline-model-1`

- Model: simple from-scratch coordinate-regression CNN
  (`build_baseline_cnn(model_id="baseline-model-1")`).
- Representation: 42 flat (x, y) keypoint coordinates in the 224-px
  input space.
- Loss: MSE on coordinates. Training metric: MAE on coordinates.
- Hyperparameters: 30 epochs, batch 32, Adam @ 1e-3, seed 42.
- Parameters: 137,962.
- Primary result: **18.10 px MPKE** on the canonical validation split
  (3,256 samples). Median 17.22 px, p90 26.32 px, p95 30.06 px,
  max-sample 75.99 px.
- Observations: serves as the from-scratch control. It predicts keypoint
  coordinates directly, so it does not preserve a spatial output map for
  localization.

### `baseline-model-2`

- Model: regularized coordinate-regression CNN
  (`build_baseline_cnn(model_id="baseline-model-2")`) based on Mikal's
  notebook baseline and standardized to the 224-px input protocol.
- Representation: 42 flat (x, y) keypoint coordinates in the 224-px
  input space.
- Loss: MSE on coordinates. Training metric: MAE on coordinates.
- Hyperparameters: defaults to 30 epochs, batch 32, Adam @ 5e-4, seed
  42, and early stopping with patience 3.
- Status: pipeline-ready, but full training and MPKE evaluation have not
  been recorded yet.

### `improved-model-1`

- Model: from-scratch residual-heatmap CNN (`build_improved_cnn`).
- Representation: 21-channel 56×56 Gaussian heatmaps (sigma 2.0).
  Predictions are decoded to keypoints in the 224-px input space with
  a sum-normalized soft-argmax (see `src/models/heatmaps.py`) so the
  same MPKE pipeline applies.
- Loss: MSE on heatmaps. Training metric: MAE on heatmaps.
- Hyperparameters: 30 epochs, batch 32, Adam @ 1e-3, seed 42, heatmap
  sigma 2.0.
- Parameters: 754,229.
- Primary result: **8.59 px MPKE** on the canonical validation split
  (3,256 samples). Median 7.43 px, p90 14.30 px, p95 17.36 px,
  max-sample 52.44 px.
- Observations: ~52% MPKE reduction vs `baseline-model-1` with ~5.5× the
  parameters. The decisive change is keeping spatial resolution at
  56×56 in the head instead of collapsing to a flat regression target,
  which preserves the localization signal the dense baseline throws
  away. Best checkpoint was saved at epoch 28.
